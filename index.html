A Framework
from
Watson Assistant
to REACT in no (development) time.
An Apache Nifi'esque visual programming solution is the goal





























$temp_ NAME
This you can use inside Watson Dialog to create temporal variables. This variables will be consumed once by the application. They can be used for flags.



$temp_ buttons
This, for example, will render a row of buttons.

$temp_ card
And this, as expected, will render a React card.

In short, the idea is to have a 1 to 1 relationship between what Watson can ask for,
and what our app is ready to perform.











RECURSION is supported.
This means you can nest statements inside other statements, and all should be good.
